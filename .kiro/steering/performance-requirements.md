# 성능 요구사항 및 최적화 전략

## 지연 시간 목표
- 전사 지연: < 2초
- 번역 지연: < 3초
- 페이지 로드: < 3초

## 오디오 전송 최적화
- base64 인코딩 지양 → WS 바이너리 프레임 사용
- 청크 크기: 20~100ms (지연 vs 오버헤드 트레이드오프)
- 16kHz mono PCM s16le 포맷

## 번역 호출 최적화
- 문장 버퍼 최대 길이 제한
- 세션당 번역 in-flight 제한 (1개) + 큐잉
- 문장 경계 감지: `. ! ?` 기준

## WebSocket 안정성
- 프론트: 자동 재연결 + 최근 이벤트 재요청
- 백엔드: 세션 상태 최소 유지 + 타임아웃 처리

## 동시 사용자
- 최소 100명 지원
- ECS Auto Scaling 설정

## 모니터링 지표
- WS 연결 수
- 오디오 처리량 (bytes/sec)
- 전사/번역 지연 (p50, p95, p99)
- 오류율
- Bedrock/Transcribe 비용 추정치

## 비용 최적화
- 음성 감지 시에만 Transcribe 활성화
- 질문 제안은 미팅 진행 속도 기준으로 업데이트(동일 화자 연속 3문장 또는 Transcribe 화자 라벨 변경)
- 컨텍스트 샘플링/요약으로 토큰 절약
